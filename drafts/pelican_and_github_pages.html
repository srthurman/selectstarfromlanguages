Running this blog on GitHub Pages with Pelican

Sun 08 March 2015
By Sara Thurman

Originally I planned on creating this blog as a Flask app, following the tutorial from Miguel Gringberg (if you have ever even thought about Flask too hard, you know that name). A little more research on hosting blogs led me to the conclustion that a static blog generator would be a better choice.

I'm not going to go into much detail on the why's, but it came down to wanting something I could easily customize, with little overhead, and free hosting options. See some of the resources I list below for further discussion.

I chose Pelican since it is well documented, both officially and with blog posts from other users, and has many themes to choose from.

I'm going to go over how I set up this blog using Pelican and hosting on GitHub Pages, and detail some trouble spots I had. I hope you find it helpful if you run into similar problems.

Initial Setup

For reference, I'm running on linux. If you're running on Windows and run into trouble feel free to contact me. I started out on Windows and run it as my other OS.

If you don't already have pip installed, do that first. You will need pip for any sort of Python development, not just this project.

Next, set up a virtual environment for your project (NOTE: The virtual environment isn't required, but strongly recommended). If you're not familiar with virtual environments, see the docs on how to install, or google until you find a tutorial that makes sense to you.

Here's how I did it:

~$ virtualenv ~/python/virtualenvs/selectstarblog
~$ cd ~/python/virtualenvs/selectstarblog
~/python/virtualenvs/selectstarblog$ source bin/activate
(selectstarblog)~/python/virtualenvs/selectstarblog$

Here's what's happening line by line:

  1. Setting up a directory to hold the files for this blog. The directory does not need to exist already.
  2. Moving into that directory, since that's where I'll be working.
  3. Activating the directory. That means it's live, and all installs I do only affect this directory.

Installing Pelican

Installing is as easy as pip install pelican. You will see a lot of other libraries being installed as well, since pelican has a number of dependencies.

Next run pelican-quickstart to get the basic structure of your site set up. You will be asked a series of questions. I used the default for most of them (in the case of y/n questions, the default is capitalized), except for:

  • Do you want to specify a URL prefix? (Y/n) n
  • Do you want to upload your website using GitHub Pages (y/N) y

I left the default for 'Is this your personal page (username.github.io)? (y/N)' since I'm running this site as a project page.

you should have the following files and directories in your folder now

bin
content
develop_server.sh
fabfile.py
include
lib
local
Makefile
output
pelicanconf.py
publishconf.py

I decided to use markdown for writing the blog, so my final setup was running pip install markdown. If you want to use reStructuredText (.rst) then you don't need to install anything else.
